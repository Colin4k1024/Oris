Oris 是什么

Oris 是一个自进化执行运行时（Self-Evolving Execution Runtime），专为 AI Agent 的推理过程设计的持久化执行内核。

核心定位

不是通用工作流引擎

管理推理过程的全生命周期 — 多步骤、非确定性决策，涉及 LLM、工具和人工介入

不是 Temporal 的替代品

专注"推理过程的持久化"：中断、恢复、回放，每一步都有明确的状态模型和契约

北极星目标

Task → Detect → Replay if trusted
     → Mutate only when needed
     → Validate → Capture → Reuse
     → Reduce reasoning over time

核心闭环：
  遇到已知问题 → 直接复用已验证经验
  遇到新问题 → 安全突变 → 验证 → 固化为基因

二阶段质量门控

Static Analysis (毫秒级, 无I/O) → LLM Critic (语义评估)

5维评分: signal_alignment × 0.30 + semantic_correctness × 0.30 + generalisability × 0.20 + test_coverage × 0.10 + complexity × 0.10

≥ 0.72 → Promote | ≥ 0.45 → ApplyOnly | < 0.45 → Reject

安全保障

• OS进程隔离 (Sandbox)
• 资源限制 (CPU/Memory/Time)
• Anti-Pattern检测 (NoOp/Hardcoded/TestDeletion)
• 置信度衰减 + 自动撤销
• Fail-closed 执行策略

分布式复用

• OEN (Oris Evolution Network) Gossip协议
• Ed25519 身份 + 签名验证
• Trust降级: Remote 信任 < Local
• 本地 Revalidation 后写入
• EVU 信用激励机制
